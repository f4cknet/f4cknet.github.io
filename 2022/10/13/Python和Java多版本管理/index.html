使用Pyenv管理Python版本

pyenv安装

安装Homebrew

安装pyenv

需要注意一下，上面会去拉github下载资源，可能会报错
curl: (7) Failed to connect to raw.githubusercontent.com port 443 after 23 ms: Couldn't connect to server
所以设置一下终端代理。我平时用的clashx，翻墙代理127.0.0.1:7890

添加到环境变量. vim ~/.zshrc

pyenv使用

1、列出可安装的Python版本

pyenv install -l

2、安装指定Python版本

pyenv install [version]

3、查看已安装Python版本

pyenv versions，其中 * 是当前使用的python版本

4、切换python版本

pyenv global 3.11.5

5、管理虚拟环境

不用虚拟环境的话，默认所有项目安装的依赖都在一个目录，比如web1项目：flask，requests和hackscript项目：cryptography、pymysql。两个项目的依赖包都安装在/Users/xxxx/.pyenv/versions/3.10.12/lib/python3.10/site-packages。想要不同项目单独管理依赖，那就必须用到python虚拟环境

6、切换到其他虚拟环境

pyenv activate [virtual_name]

7、退出虚拟环境

Pyenv deactivate

使用Jenv管理jdk

jenv安装

jenv使用

1、安装不同JDK

jenv和pyenv不一样，pyenv可以直接通过pyenv install [version]安装对应版本的python，而jenv只是管理，安装jdk需要另外独立操作。

先brew search openjdk查询一下当前可安装的jdk版本

然后执行,分别安装jdk17和jdk1.8

brew search查看已经安装的JDK，打勾绿色的是已经安装的

2、将不同版本JDK加入jenv中管理

使用jenv add命令将不同java版本添加到jenv进行管理

Mac中jdk路径正常都在/Library/Java/JavaVirtualMachines/

3、切换jdk版本

jenv global [version]

4、支持mvn

如果环境变量里没有设置JAVA_HOME变量的话，mvn会报错

而我们使用jenv来管理，注定就不能在环境变量中添加JAVA_HOME，这样系统默认就使用JAVA_HOME中的jdk了，就无法用jenv来管理不同版本的jdk环境了。

执行命令：jenv enable-plugin maven就可以了
